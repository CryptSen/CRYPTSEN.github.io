Cross-Ledger Cryptocurrency Analytics Platform

GraphSense is an open source platform for analyzing cryptocurrency transaction ledgers. At the moment, it supports analysis of Bitcoin, Bitcoin Cash, Litecoin, and Zcash transactions.

Features

  • Search by address, tag, transaction or block in several cryptocurrency ledgers.
  • Navigate in transaction network abstractions computed from various ledgers and inspect statistical properties of nodes (addresses, clusters) and edges (aggregated transactions).
  • Automatically search for transaction paths connecting two nodes.

  • BlockSci integration: GraphSense makes use of BlockSci for parsing blockchains, obtaining exchange rates, and filtering CoinJoins.

  • Support for data-driven analytics via the GraphSense REST-API

  • Horizontal Scalability: cryptocurrency blockchains are growing and new currencies appear on the horizon. GraphSense should be future-proof, because it is built on Apache Spark and Cassandra for horizontal scalability.

Technical Architecture

GraphSense is built on scalable and distributed cluster technology and therefore requires a number of software components. They must be setup and/or executed in the following order:

  • btc-client, bch-client, ltc-client, zec-client: docker containers encapsulating the most-recent cryptocurrency client versions

  • graphsense-blocksci: a dockerized component to synchronize BlockSci data to Apache Cassandra

  • graphsense-transformation: an Apache Spark ETL pipeline for computing statistics and network representations from raw blockchain data stored in Cassandra

  • graphsense-REST: an API for retrieving data from the underlying Cassandra store

  • graphsense-tagpacks: a data structure for collecting cryptocurrency attribution tags with associated provenance and categorization metadata

  • graphsense-dashboard: a user-interface allowing search, inspection, and traversal of cryptocurrency entities
